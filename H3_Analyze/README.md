3 — Analyze Phase
Health Holding Company | Emergency Department DMAIC Project

The Analyze phase answers one critical question:

“What is causing Emergency Department overcrowding, delays, and extended patient waiting times?”

Folder Structure 📂

| Subfolder | Tool | Purpose |
|------------|------|----------|
| 01_Fishbone_Diagram/ | Ishikawa / Fishbone | Identify all possible causes of delays and overcrowding |
| 02_5_Why_Analysis/ | 5-Why | Drill down from symptoms to root causes |
| 03_Hypothesis_Testing/ | T-test, ANOVA, Chi-Square | Statistically confirm significant causes |
| 04_Lean_Waste_Analysis/ | 8 Wastes (DOWNTIME) | Identify operational waste in patient flow |
| 05_Process_Bottleneck_Analysis/ | Bottleneck Analysis | Identify congestion points across the ED workflow |
| 06_Root_Cause_Summary/ | Summary Report | Consolidate all confirmed root causes |

Lean Thinking in the Analyze Phase 🧠

Lean and Six Sigma work together:

Six Sigma confirms root causes statistically
Lean identifies operational waste and inefficiencies

The Emergency Department project combines both approaches to improve patient flow and reduce delays.

The 8 Wastes — DOWNTIME Methodology 🏥

| Letter | Waste | Description | Emergency Department Example |
|--------|--------|-------------|-------------------------------|
| D | Defects | Errors and rework | Incorrect triage classification |
| O | Overproduction | Doing more than needed | Repeated documentation steps |
| W | Waiting | Delays and idle time | Patients waiting for physician assessment |
| N | Non-Utilized Talent | Underusing staff skills | Nurses not involved in workflow decisions |
| T | Transport | Unnecessary movement | Patients transferred multiple times |
| I | Inventory | Excess WIP / backlog | Long waiting queues in triage |
| M | Motion | Unnecessary movement of people | Staff walking between distant stations |
| E | Extra Processing | Additional unnecessary work | Duplicate patient registration activities |

Key Principles 📌

Principle 1: Separate Symptoms from Causes

| Symptom | Root Cause |
|----------|-------------|
| Door-to-Doctor Time = 95 min | Minor cases occupying urgent-care capacity |
| Long triage delays | Inconsistent staffing during peak hours |
| Overcrowding | Slow admissions and discharge bottlenecks |
| High LWBS rate | Excessive waiting times and poor queue visibility |

Never treat the symptom itself as the root cause.

Principle 2: Let Data Confirm — Not Opinion

Every suspected root cause must be validated statistically.

p-value ≤ 0.05 = statistically significant

“We think evenings are worse” → Hypothesis

Data analysis confirms whether it is true


Principle 3: Combine Lean + Six Sigma

| Six Sigma Approach | Lean Approach | Combined Benefit |
|--------------------|---------------|------------------|
| Statistical validation | Waste identification | Data-driven operational improvement |
| Hypothesis testing | Patient-flow analysis | Accurate root-cause confirmation |
| KPI analysis | Workflow optimization | Sustainable process improvement |

Common Mistakes ⚠️

| Mistake | Impact | Prevention |
|----------|--------|------------|
| Jumping directly to solutions | Wrong improvements implemented | Complete root-cause validation first |
| Using only one analysis tool | Missed causes | Combine Fishbone, 5-Why, and statistics |
| Ignoring Lean waste | Workflow inefficiencies remain | Perform Lean waste analysis |
| Treating assumptions as facts | Incorrect decisions | Validate with operational data |
| Not involving frontline staff | Missed practical insights | Include physicians and nurses |
| Ignoring variation between shifts | Inconsistent improvements | Analyze shift-level performance |
